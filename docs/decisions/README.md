# Architectural Decisions

This directory records significant architectural and debugging workflow decisions using a lightweight ADR (Architecture Decision Record) format.

## When to add a record

Create a new file here when:
- Choosing a library, framework, or pattern over alternatives
- Making a trade-off that affects debugging, performance, or agentic velocity
- Defining conventions that agents must follow when working in this repo

## Format

Keep each record concise. Include:
- **Decision:** What was decided
- **Context:** Why this decision was needed
- **Consequences:** What this means for future work and agent workflows
