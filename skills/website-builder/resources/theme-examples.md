## Selection Heuristic

When the user does not specify a look:

1. Match the business `type` (from `BusinessProfile.type`) to one of the subcategories above.
2. Pick the `heroStyle` and `cardStyle` from the reference that aligns with 2 of the user's `vibe.adjectives`.
3. Use the **palette** section as source material for colour selection. Aim for two distinct directions:
   - **Direction A** should lean toward the more dominant vibe adjective.
   - **Direction B** should lean toward the second adjective, or the inverse (e.g. if A is *warm*, B is *cool*).
4. If the user uploaded a logo or hero image, extract the dominant hue and use it as `primary`. Use a complementary or analogous colour for `secondary`.
5. If the user has an existing website URL, fetch the homepage and sample 2–3 colours from the rendered CSS as the starting palette.

## Notes

- All colour values above are hex. Convert to the theme JSON format exactly as shown.
- Typography names should match web-safe font families available via `next/font/google` or already bundled.
- Keep accessibility in mind: maintain WCAG AA contrast between `text` and `background`, and between `primary` text on `secondary` surfaces.
- Dimension spec bundle names should use the chosen direction keyword (e.g. `"Rustic Warmth"`, `"Cool Minimal"`).
- **Reference links:** Use these to ground theme generation in authoritative, current (2026) real-world design. When generating themes for a given subcategory, fetch at least one of the listed links, identify the dominant design patterns (palette, typography, card style, hero treatment), and use those to seed or validate the chosen direction.
